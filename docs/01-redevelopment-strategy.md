# Prairie Rail Solutions - Redevelopment Strategy

## 1) Primary Objective
Increase qualified inbound quote requests from industrial operators, grain facilities, and short-line partners by improving trust, clarity, and conversion flow.

## 2) Current Site Observations
- Platform: Wix
- Core pages exist and return 200 status.
- Service offering is strong but messaging structure is inconsistent.
- Some copy formatting/capitalization mismatches reduce perceived professionalism.
- CTA forms exist but proof and trust support near CTAs can be improved.

## 3) Target User Segments
- Grain elevator operators and managers
- Industrial facility operations managers
- Short-line rail operations teams
- Procurement and compliance stakeholders

## 4) Value Proposition Framework
- Safety and compliance first
- Reliable execution by rail-experienced teams
- Practical, field-tested support for first/last mile and maintenance needs

## 5) Positioning Statement
Prairie Rail Solutions is the dependable rail operations partner for prairie-region industry, delivering compliant, safety-first inspections, maintenance, and switching support that keeps supply chains moving.

## 6) Conversion Goals
- Primary: Quote/contact form submissions
- Secondary: Direct call clicks and email clicks
- Tertiary: Job applications (careers funnel)

## 7) KPI Targets (First 90 Days Post-Launch)
- +30% increase in qualified form submissions
- +20% increase in time on service pages
- -25% drop in homepage bounce rate
- >= 95 score for SEO and Best Practices on core pages (Lighthouse)

## 8) Brand/Content Priorities
- Standardize heading styles and capitalization
- Tighten service descriptions around outcomes
- Add trust blocks near all major CTAs:
  - Certifications
  - Safety/compliance standards
  - Industry memberships
  - Process clarity (what happens after inquiry)

## 9) Technical Recommendation
Use Next.js (App Router) with a headless CMS for maintainable growth.

Suggested stack:
- Frontend: Next.js + TypeScript + Tailwind
- Content: Sanity or Contentful
- Forms: server actions + email/CRM integration
- Analytics: GA4 + Search Console + event tracking
- Hosting: Vercel

## 10) Launch Risk Controls
- Preserve existing slugs where possible
- 301 redirect map for changed URLs
- Migrate metadata per page
- Validate structured data and sitemap before go-live
- Run production checks on desktop and mobile
